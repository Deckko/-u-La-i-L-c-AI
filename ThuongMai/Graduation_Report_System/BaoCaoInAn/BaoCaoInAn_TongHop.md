Nhiệm vụ trọng tâm của đợt thực tập này là giúp học sinh tiếp cận môi trường làm việc chuyên nghiệp của doanh nghiệp phần mềm, củng cố và vận dụng các kiến thức lập trình Web (Next.js, Laravel API, Qdrant Vector Database) đã được trang bị tại nhà trường để xây dựng hoàn chỉnh hệ thống: **"Website bán hàng trực tuyến tích hợp AI Chatbot tư vấn khách hàng tự động và Semantic Search"**.

Các công việc cụ thể được giao bao gồm:
1. Tìm hiểu quy trình làm việc Agile/Scrum của các nhóm dự án tại doanh nghiệp.
2. Khảo sát nghiệp vụ quản trị kho, xử lý đơn đặt hàng và tương tác chăm sóc khách hàng.
3. Thiết kế cơ sở dữ liệu quan hệ PostgreSQL và cấu hình Qdrant Vector Database.
4. Lập trình giao diện Frontend (Next.js 16 App Router) hỗ trợ responsive đa thiết bị và song ngữ i18n.
5. Xây dựng APIs nghiệp vụ Backend (Laravel PHP) và tích hợp dịch vụ AI RAG Engine (FastAPI).
6. Triển khai xuất bản dự án tĩnh (Static HTML Export) lên dịch vụ Firebase Hosting của Google.
7. Thực hiện kiểm thử tích hợp hệ thống, rà soát và khắc phục các lỗ hổng bảo mật liên quan đến thông tin xác thực thô trong mã nguồn.

Học sinh thực tập có trách nhiệm thực hiện đúng tiến độ được phân công, ghi chép nhật ký thực tập đầy đủ mỗi tuần và hoàn thành cuốn báo cáo tốt nghiệp đạt chất lượng kỹ thuật theo quy định của nhà trường.

# PHẦN 1: LỜI NÓI ĐẦU

Trong những năm gần đây, sự phát triển bùng nổ của mạng lưới Internet và các công nghệ di động đã làm thay đổi hoàn toàn thói quen tiêu dùng của xã hội. Thương mại điện tử (E-commerce) từ một xu hướng công nghệ mới đã trở thành hạ tầng bắt buộc đối với mọi doanh nghiệp bán lẻ muốn tồn tại và phát triển. Tuy nhiên, đi kèm với sự phát triển đó là yêu cầu ngày càng cao của người tiêu dùng về mặt trải nghiệm mua sắm số hóa: tốc độ tải trang phải tức thì, giao diện trực quan và đặc biệt là hệ thống tư vấn tìm kiếm thông tin sản phẩm phải cực kỳ thông minh.

Qua thời gian khảo sát thực tế và cọ xát trực tiếp tại **Công ty Cổ phần SweetSoft**, em nhận thấy hầu hết các website bán hàng trực tuyến hiện nay tại Việt Nam đều đang gặp phải hai rào cản công nghệ lớn:
1. **Hệ thống tìm kiếm cứng nhắc**: Công cụ tìm kiếm cũ dựa trên so khớp từ khóa thô (Keyword Matching). Khi khách hàng nhập sai chính tả hoặc mô tả nhu cầu bằng ngôn ngữ giao tiếp đời thường (ví dụ: *"tôi muốn tìm một chiếc áo phông mát mẻ mặc mùa hè"*), hệ thống hoàn toàn bất lực không thể đưa ra kết quả gợi ý phù hợp.
2. **Quá tải dịch vụ chăm sóc khách hàng trực tuyến**: Bộ phận nhân sự tư vấn thường xuyên bị quá tải vào các giờ cao điểm hoặc ban đêm, dẫn tới việc phản hồi chậm trễ và đánh mất trực tiếp khách hàng tiềm năng.

Để giải quyết triệt để các hạn chế trên, em đã lựa chọn đề tài thực tập tốt nghiệp: **"Thiết kế và xây dựng website trực tuyến tích hợp AI Chatbot và truy vấn ngữ nghĩa Semantic RAG"** cho thương hiệu thời trang cao cấp DECKKO Classic. Đồ án tập trung nghiên cứu ứng dụng kiến trúc 3 lớp phân rã dịch vụ (Decoupled Architecture) kết hợp sức mạnh hiển thị tốc độ cao của **Next.js 16 App Router** phía Client, tính năng quản trị cơ sở dữ liệu giao dịch an toàn của **Laravel API (PHP)** kết hợp **PostgreSQL** ở Backend, và dịch vụ **AI RAG Search Engine (Python FastAPI)** sử dụng cơ sở dữ liệu vector **Qdrant DB**.

Mục tiêu cốt lõi của đồ án là giúp khách hàng tìm kiếm sản phẩm chính xác bằng ngôn ngữ tự nhiên, giao tiếp trực quan với chatbot trợ lý AI để hỏi đáp size và chất liệu mà không bị lỗi ảo giác (Hallucination), đồng thời xây dựng một trang quản trị Admin mạnh mẽ cập nhật doanh thu thực tế và quản lý kho hàng an toàn tuyệt đối chống tranh chấp luồng dữ liệu (Race Condition).

#### 2.1.1 Cơ cấu tổ chức và bộ phận kỹ thuật
SweetSoft tổ chức nhân sự theo các phòng ban chức năng chuyên biệt:
* **Bộ phận lập trình Frontend**: Chịu trách nhiệm thiết kế giao diện, tối ưu hóa trải nghiệm người dùng (UI/UX) sử dụng các framework hiện đại như React, Next.js, Vue.js.
* **Bộ phận lập trình Backend**: Chịu trách nhiệm xây dựng API, xử lý logic nghiệp vụ phía máy chủ sử dụng Node.js, PHP Laravel, Python Django và Java Spring Boot.
* **Bộ phận Quản trị Cơ sở dữ liệu & DevOps**: Thiết lập cấu hình hệ thống, quản trị PostgreSQL, MySQL, Redis và cơ sở dữ liệu vector Qdrant trên nền tảng Docker/Kubernetes.
* **Bộ phận kiểm thử phần mềm (QA/QC)**: Thực hiện kiểm thử hộp đen, kiểm thử hộp trắng và kiểm thử hiệu năng tự động (Load Test/Stress Test).

### 2.3 Phân tích thiết kế hệ thống và Cơ sở dữ liệu quan hệ

#### 2.3.1 Sơ đồ ERD Schema PostgreSQL
Cơ sở dữ liệu quan hệ PostgreSQL của hệ thống được chuẩn hóa để đảm bảo tốc độ truy vấn nhanh nhất và tính toàn vẹn khóa ngoại tuyệt đối:

```sql
-- 1. BẢNG NGƯỜI DÙNG (users)
CREATE TABLE users (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    email VARCHAR(150) UNIQUE NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    phone_number VARCHAR(20) NULL,
    created_at TIMESTAMPTZ DEFAULT CURRENT_TIMESTAMP NOT NULL
);

-- 2. BẢNG DANH MỤC & THƯƠNG HIỆU
CREATE TABLE categories (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    name VARCHAR(100) NOT NULL,
    slug VARCHAR(120) UNIQUE NOT NULL
);

CREATE TABLE brands (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    name VARCHAR(100) NOT NULL,
    slug VARCHAR(120) UNIQUE NOT NULL
);

-- 3. BẢNG SẢN PHẨM & BIẾN THỂ KHO HÀNG (variants)
CREATE TABLE products (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    brand_id UUID REFERENCES brands(id) ON DELETE SET NULL,
    category_id UUID REFERENCES categories(id) ON DELETE SET NULL,
    name VARCHAR(150) NOT NULL,
    slug VARCHAR(180) UNIQUE NOT NULL,
    description TEXT NULL
);

CREATE TABLE product_variants (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    product_id UUID REFERENCES products(id) ON DELETE CASCADE NOT NULL,
    sku VARCHAR(100) UNIQUE NOT NULL,
    price DECIMAL(19, 4) NOT NULL,
    stock_qty INTEGER DEFAULT 0 NOT NULL,
    attributes JSONB NOT NULL DEFAULT '{}'
);

-- 4. BẢNG ĐƠN HÀNG VÀ HÓA ĐƠN CHI TIẾT
CREATE TABLE orders (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    order_code VARCHAR(64) UNIQUE NOT NULL,
    user_id UUID REFERENCES users(id) ON DELETE RESTRICT NOT NULL,
    total_amount DECIMAL(19, 4) NOT NULL,
    status VARCHAR(32) DEFAULT 'pending' NOT NULL,
    created_at TIMESTAMPTZ DEFAULT CURRENT_TIMESTAMP NOT NULL
);

CREATE TABLE order_items (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    order_id UUID REFERENCES orders(id) ON DELETE CASCADE NOT NULL,
    variant_id UUID REFERENCES product_variants(id) ON DELETE RESTRICT NOT NULL,
    quantity INTEGER NOT NULL CHECK (quantity > 0),
    unit_price DECIMAL(19, 4) NOT NULL
);
```

#### 2.3.2 Đặc tả các bảng dữ liệu chủ chốt
* **Bảng `products`**: Lưu trữ thông tin sản phẩm cha để làm gốc. Chứa các trường liên kết khóa ngoại trỏ sang bảng danh mục (`category_id`) và thương hiệu (`brand_id`). 
* **Bảng `product_variants`**: Lưu trữ các phiên bản biến thể vật lý thực tế của sản phẩm. Mỗi biến thể có mã định danh kho hàng duy nhất `sku`, giá tiền và số lượng tồn kho `stock_qty` riêng biệt. Sử dụng kiểu dữ liệu `JSONB` của PostgreSQL giúp mở rộng các thuộc tính động (Màu sắc, kích cỡ, độ co giãn) dễ dàng mà không làm phình to cấu trúc bảng.
* **Bảng `orders` & `order_items`**: Lưu vết thông tin thanh toán của khách hàng. Ràng buộc khóa ngoại `ON DELETE RESTRICT` được thiết lập trên liên kết người dùng để ngăn chặn việc xóa tài khoản khách hàng khi lịch sử hóa đơn giao dịch của họ vẫn tồn tại trên hệ thống.

##### Thiết lập cơ chế kiểm soát lỗi Runtime hệ thống (ErrorBoundary.tsx)
Để ngăn chặn hoàn toàn hiện tượng sụp đổ giao diện (White Screen) khi gặp lỗi biên dịch runtime không mong muốn, em đã phát triển tệp tin [ErrorBoundary.tsx](file:///c:/antigravity/ThuongMai/nextjs-frontend/src/components/atoms/ErrorBoundary.tsx) bao bọc toàn bộ mã nguồn ứng dụng ở cấp độ cao nhất:

```tsx
'use client';

import React from 'react';
import { AlertTriangle, RefreshCcw } from 'lucide-react';

interface Props {
  children: React.ReactNode;
  fallback?: React.ReactNode;
}

interface State {
  hasError: boolean;
  error?: Error;
}

export class ErrorBoundary extends React.Component<Props, State> {
  constructor(props: Props) {
    super(props);
    this.state = { hasError: false };
  }

  static getDerivedStateFromError(error: Error): State {
    return { hasError: true, error };
  }

  componentDidCatch(error: Error, info: React.ErrorInfo) {
    console.error('[ErrorBoundary]', error, info.componentStack);
  }

  render() {
    if (this.state.hasError) {
      if (this.props.fallback) return this.props.fallback;
      return (
        <div className="min-h-[40vh] flex flex-col items-center justify-center gap-4 p-8 text-center">
          <div className="w-16 h-16 rounded-full bg-red-500/10 border border-red-500/30 flex items-center justify-center">
            <AlertTriangle className="w-8 h-8 text-red-400" />
          </div>
          <div className="space-y-2">
            <h2 className="text-lg font-black text-zinc-50 uppercase tracking-wider">Đã xảy ra lỗi</h2>
            <p className="text-sm text-zinc-500 max-w-sm">{this.state.error?.message || 'Vui lòng thử lại.'}</p>
          </div>
          <button onClick={() => this.setState({ hasError: false, error: undefined })}
                  className="flex items-center gap-2 bg-amber-500 hover:bg-amber-600 text-zinc-950 px-6 py-2.5 font-black text-xs uppercase tracking-widest rounded-sm">
            <RefreshCcw className="w-3.5 h-3.5" /> Thử lại
          </button>
        </div>
      );
    }
    return this.props.children;
  }
}
```

#### 2.5.1 Xử lý transaction trừ kho chống Race Condition
Dưới đây là đặc tả logic nghiệp vụ của [CheckoutService.php](file:///c:/antigravity/ThuongMai/laravel-backend/app/Services/CheckoutService.php) thực thi đặt hàng transactional. ```php
$orderInfo = DB::transaction(function () use ($cartItems, $data, $userId) {

        $variant = ProductVariant::lockForUpdate()->find($item['variant_id']);

        // Trừ kho vật lý
        $variant->decrement('stock_qty', $item['quantity']);
        $totalAmount += $variant->price * $item['quantity'];
    }

### 2.6 Tích hợp AI RAG Engine và Vector Database Qdrant

#### 2.6.1 Giải pháp tìm kiếm ngữ nghĩa Semantic Search
Hệ thống giải quyết bài toán tư vấn bằng mô hình nhúng (Word Embedding) 1536 chiều của OpenAI.

```
[Dữ liệu Postgres] ➔ [Tạo văn bản thô đặc tả] ➔ [Embedding 1536d] ➔ [Lưu vào Qdrant DB]
                                                                        │
[Câu hỏi khách hàng] ➔ [Embedding 1536d] ➔ [Cosine Similarity] ➔ [Lọc điểm > 0.72]
                                                                        │
[Tổng hợp prompt chứa context chuẩn xác] ➔ [Gửi LLM GPT-4o-mini] ➔ [Trả lời khách hàng]
```

Cơ chế này loại bỏ hoàn toàn hiện tượng ảo giác (bịa đặt thông tin sản phẩm và giá cả) của mô hình ngôn ngữ lớn, đảm bảo câu trả lời luôn bám sát đặc tính sản phẩm thực tế của DECKKO.

#### 2.6.2 Thuật toán Cosine Similarity trong chatbot
Độ tương đồng ngữ nghĩa giữa câu hỏi của khách hàng (Vector $\mathbf{A}$) và mô tả chi tiết của sản phẩm thời trang (Vector $\mathbf{B}$) được đo lường bằng thuật toán Cosine Similarity:

$$\text{Cosine Similarity} = \frac{\mathbf{A} \cdot \mathbf{B}}{\|\mathbf{A}\| \|\mathbf{B}\|} = \frac{\sum_{i=1}^{n} A_i B_i}{\sqrt{\sum_{i=1}^{n} A_i^2} \sqrt{\sum_{i=1}^{n} B_i^2}}$$

Nếu giá trị Cosine Similarity đạt điểm số trên `0.72`, sản phẩm đó sẽ được trích xuất làm ngữ cảnh (Context) đáng tin cậy để đưa vào Prompt gửi cho LLM. Nhờ đó chatbot AI có khả năng giải đáp chuẩn xác câu hỏi của người dùng mà không bị ảo giác thông tin.
